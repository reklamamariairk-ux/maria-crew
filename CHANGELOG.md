# Maria Crew — журнал изменений

## Май 2026 — аудит логики и фиксы

### Исправлено
- **Награда за квиз: +1 монета за правильный ответ** (до +5 за идеальный квиз). Сервер всегда начислял +1, но тексты в Mini App и напоминалки в канал обещали +2/+10. Привёл все тексты — Mini App, `/coins`, `remindQuiz` — в соответствие с реальной механикой +1/+5.
- **Защита от двойного начисления за квиз.** Добавлена миграция `027_quiz_attempt_unique.sql` с UNIQUE-индексом по `(employee_id, question_id, иркутская дата)`. `submitAnswer` использует `ON CONFLICT DO NOTHING` и не начисляет монеты повторно. Ранее можно было ответить на 3 вопроса, закрыть Mini App, открыть и снова ответить на те же — получая лишние монеты.
- **`getDailyQuestionsWithAnswers` фильтрует уже отвеченные сегодня вопросы** — после повторного открытия Mini App квиз продолжается с того места, где остановились, а не показывает вопросы заново.
- **Сезонный челлендж теперь авто-завершается.** Раньше `getActiveChallenge` показывал прогресс, но `checkAndCompleteChallenge` дёргался только из webapp endpoint, который никто не вызывал. Теперь проверка условий выполняется при каждом GET `/api/webapp/challenge`.
- **Полный набор лейблов для причин монет в Mini App.** Добавлены `training_meeting`, `knowledge_applied`, `bad_review`, `dirty_store`, `training_resistance` — раньше показывалось сырое имя.
- **`/coins` в боте.** В блок «Как заработать монеты» добавлены квиз и ежедневный вход.
- **Cache-bust для `webapp/main.js`.**

## Последняя крупная итерация (апрель 2026)

### Новые функциональные возможности

**Многопользовательская админка с ролями**
- Логин по `username + password` (вместо общего секрета)
- 3 роли: `superadmin` / `editor` («Админище») / `coin_admin` («Администратор»)
- Bootstrap суперадмина: первый запуск создаёт `admin / $ADMIN_SECRET`
- Раздел «Доступы» (только суперадмин): создание/удаление админов, смена ролей и паролей
- Принудительная смена пароля при первом входе после сброса (флаг `must_change_password`)
- Каждый админ может сменить свой пароль через иконку 🔑 в сайдбаре
- Rate limit на `/api/auth/login`: 10 попыток / 15 минут с IP

**Маршруты по ролям:**
- `/api/coins/award`, `/api/employees/bulk-coins` → `superadmin + coin_admin`
- `/api/admin-users/*`, `PUT /api/config/mvp` → только `superadmin`
- Остальные write-ручки → запрещены для `coin_admin`
- Frontend: `tabAllowed(tab, role)` — единая точка истины (`admin/app.js`)

**Сбор номера телефона сотрудников (для 1С)**
1. Mini App: `tg.requestContact()` с предупреждением «для начисления подарков в 1С»
2. Бот: reply-keyboard «📱 Поделиться номером» на `/start` если телефон отсутствует
3. Админка: ручное поле в карточке сотрудника

**Награды MVP — настраиваются админом**
- `mvp_coin_reward` (default 50) — лучшему сотруднику месяца
- `top_store_coin_reward` (default 30) — каждому в топ-точке
- Начисляются при «Обработать месяц», идемпотентно

**Управление героями**
- Создание/удаление героев в админке
- Загрузка картинок через Cloudinary (unsigned upload preset)
- Heroes имена изменяются inline в таблице

**Управление челленджами**
- Удаление челленджа в админке (CASCADE удаляет участников)
- Корректное отображение статуса: Активен / Запланирован / Завершён / Отключён

**Дашборд**
- MVP считается «на лету» из последних метрик (не требует «Обработать месяц»)
- Бейдж количества `pending` заявок в сайдбаре (опрос раз в 2 мин)
- График вовлечённости показывает все 30 дней (включая пустые), сегодня подсвечен
- Месячные счётчики корректно работают на границе UTC через Иркутск (UTC+8)

**Maria Store**
- Цены за монеты увеличены ×10 (Кофе 50м → Сертификат 500м)
- v3.0 каталог призов (миграция 018)
- Race-protection при быстрых обменах (`storeRequestId`)

**Экспорт CSV**
- `GET /api/coins/export?from=&to=&storeId=` → файл с BOM для Excel
- Кнопка во вкладке «Монеты»

**Plot UX**
- Нижнее меню в Mini App стало плавающей «пилюлей» (отступы, скругление, тень)
- Активный таб подсвечивается точкой над иконкой

- Race-protection в webapp loadStore
