## 1. Какие данные собираем

Maria Crew — внутренний инструмент мотивации сотрудников кондитерских «Мария».
Приложение не предназначено для общего использования.

Собираемые данные:
- **ФИО** — для идентификации сотрудника
- **Телефон** — для связи и для интеграции с 1С (начисление премий)
- **Telegram ID и username** — для уведомлений через бот
- **Фото профиля** (опционально) — берётся из Telegram-аккаунта
- **Точка работы** — конкретная кондитерская
- **История активности**: чек-ины, ответы в квизе, начисления/списания монет, выданные карточки, заявки на призы

Push-токены (FCM/APNs) — только для отправки уведомлений из приложения.

## 2. Как используем

- **Логин и идентификация** — телефон + одноразовый PIN из Telegram
- **Уведомления** — Telegram-бот и push в мобильном приложении
- **Внутренние отчёты** — рейтинги сотрудников и точек, аналитика квиза
- **Бухгалтерия** — синхронизация с 1С для начисления премий

## 3. Где храним

База данных PostgreSQL (Neon, регион EU). Бэкенд работает на Render (регион EU/US).
Файлы (фото героев) — Cloudinary.

Доступ к данным имеют только администраторы вашей организации. Anthropic / OpenAI /
другие AI-сервисы данные не получают.

## 4. Срок хранения

- Активный сотрудник: пока работает + 6 месяцев после увольнения (для отчётности).
- Удалённый аккаунт: личные данные (имя, телефон, Telegram-привязка) удаляются
  немедленно. История транзакций обезличивается, но сохраняется для бухгалтерии.

## 6. Третьи стороны

- **Telegram** (политика: https://telegram.org/privacy) — авторизация и уведомления
- **Firebase Cloud Messaging** (политика: https://firebase.google.com/support/privacy) — push в мобильное приложение
- **Cloudinary** (политика: https://cloudinary.com/privacy) — хранение фото героев (без личных данных)
- **Render**, **Neon** — хостинг

## 7. Безопасность

- Пароли — нет (только PIN, действует 10 минут)
- JWT-токены — HMAC-SHA256, действуют 30 дней
- Все запросы — HTTPS
- БД — TLS-only соединение
